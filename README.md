## 🛠 기술적 구현

### 🤖 AutoGluon 선택 이유
**"첫 팀 프로젝트에서 빠른 결과가 필요했어요"**

1. **학습의 효율성**
   - 복잡한 하이퍼파라미터 튜닝 자동화
   - 다양한 모델 자동 비교 및 선택
   - 팀원들이 모델링보다는 문제 해결에 집중할 수 있게 도움

2. **우수한 성능**
   - MAE 3.89로 다른 모델 대비 최고 성능
   - 앙상블 모델로 안정적인 예측 결과

3. **실용적 접근**
   - 공모전 일정에 맞는 빠른 개발
   - 실제 배포 가능한 모델 생성

### 📊 활용 데이터 및 전처리

**다양한 데이터 소스 통합:**

1. **대중교통 데이터**
   - 승차/하차 수요 (Target: RIDE_DEMAND)
   - 시간대별, 격자별 이용 패턴

2. **외부 데이터 통합**
   - **정류장 데이터**: 격자별 버스정류장 개수
   - **국토 데이터**: 인구, 빌딩 수, 용적률
   - **날씨 데이터**: 온도, 강수량

3. **팀워크로 완성한 데이터 파이프라인**
   - 격자 단위 데이터 통합
   - 시간대별 매칭 및 결측치 처리
   - 각 팀원의 전문 분야별 데이터 검증

## 📈 모델 성능 및 분석

### 🏆 최종 모델 성능
- **WeightedEnsemble_L2**: MAE 3.77 (검증) → 3.89 (실제 예측)
- **성능 개선 과정**:
  1. 초기 예측: MAE 3.95
  2. 음수값 제거: MAE 3.93
  3. 정수 변환: MAE 3.89

### 📊 특성 중요도 분석
**팀 분석 결과:**
1. **하차 수요**: 33% (최고 영향력)
2. **격자 ID**: 17% (공간적 특성)
3. **시간대**: 14% (시간적 패턴)
4. **공휴일**: 2%

### 🎯 핵심 발견점
- **하차-승차 수요 강한 상관관계** (0.91)
- **지역별 특성이 수요에 큰 영향**
- **시간대별 뚜렷한 패턴 존재**
- **날씨보다는 공간-시간 요인이 더 중요**

**"혼자서는 절대 해낼 수 없었던 일"**

```
Daejeon-Public-Transport-Demand-Prediction/
```
